## 二、智能体执行指令

> 将以下内容直接发送给 AI 智能体，智能体会按顺序执行每一步。

```
请按以下步骤配置本机的 Git 开发环境：

1. 检查 Git 是否安装（git --version），未安装则提示我

2. 配置用户信息：
   git config user.name "我的GitHub用户名"
   git config user.email "我的邮箱"

3. 添加远程仓库：
   git remote add origin https://github.com/luojiang419/ee2.git
   （如果已存在则跳过）

4. GitHub 授权登录（需要先安装 gh CLI）：
   gh auth login
   选择 GitHub.com → HTTPS → Yes → 浏览器登录

5. 配置代理和 SSL（如果本机需要代理才能访问 GitHub）：
   git config --global http.proxy http://127.0.0.1:7890
   git config --global https.proxy http://127.0.0.1:7890
   git config --global http.sslBackend schannel

6. 拉取最新代码并覆盖本地：
   git fetch origin
   git checkout main
   git reset --hard origin/main

7. 设置分支追踪：
   git branch --set-upstream-to=origin/main main

8. 验证：git remote -v 和 git status 确认一切正常

9. 完成后提示我：在 VS Code Source Control 面板中，以后修改代码后写提交说明，然后点击 Sync Changes 即可一键推送。
```

## 三、常见问题排查

### 拉取/推送失败：无法连接 GitHub

**错误信息：**
```
Failed to connect to github.com port 443 after 21000 ms: Could not connect to server
```

**原因：** 本机网络无法直连 GitHub，需要配置代理。

**解决方法：**
```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
git config --global http.sslBackend schannel
```

### 拉取失败：SSL 连接中断

**错误信息：**
```
error: RPC failed; curl 56 OpenSSL SSL_read: unexpected eof while reading
fatal: early EOF
```

**原因：** SSL 后端使用了 `openssl`，与代理不兼容。需切换为 `schannel`。

**解决方法：**
```bash
git config --global http.sslBackend schannel
```

## 四、仓库权限（仓库所有者操作）

在 GitHub 上为协作者添加写入权限：

1. 打开 https://github.com/luojiang419/ee2
2. **Settings** → **Collaborators** → **Add people**
3. 输入协作者的 GitHub 用户名
4. 协作者会收到邀请邮件，接受后即可推送代码
