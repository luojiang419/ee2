# EE2X 静态更新协议

## 目录结构

```text
{remoteRoot}/
└── updates/
    └── stable/
        ├── latest.json
        └── releases/
            └── {releaseId}/
                ├── launcher/
                │   ├── EE2X-launcher-{releaseId}.zip
                │   └── release-manifest.json
                ├── game/
                │   ├── EE2X-game-{releaseId}.zip
                │   └── release-manifest.json
                └── release-notes.txt
```

## latest.json

字段：

- `schemaVersion`
- `channel`
- `version`
- `releaseNotes`
- `required`
- `manifestUrl`
- `packageUrl`
- `packageSha256`
- `packageSize`
- `publishedAt`
- `packages.launcher.manifestUrl`
- `packages.launcher.packageUrl`
- `packages.launcher.packageSha256`
- `packages.launcher.packageSize`
- `packages.game.manifestUrl`
- `packages.game.packageUrl`
- `packages.game.packageSha256`
- `packages.game.packageSize`

说明：

- `manifestUrl / packageUrl / packageSha256 / packageSize` 继续保留为旧版客户端兼容字段，始终指向 `game` 包。
- 新版启动器优先读取 `packages.launcher` 与 `packages.game`。
- `packages.launcher` 仅在 launcher manifest 存在实际变更时下发。
  - 判定条件：`launcher/release-manifest.json` 中 `files` 非空。
  - 若 launcher 只是空包（`files=[]`），服务端仍可存档该 package，但 `latest.json` 不再暴露 `packages.launcher`。

## release-manifest.json

字段：

- `schemaVersion`
- `version`
- `rootDirName`
- `packageFileName`
- `packageSha256`
- `applyMode`
- `protectedPaths`
- `deleteList`
- `files`

`files` 每项包含：

- `path`
- `size`
- `sha256`

## 应用规则

1. `game` 包 ZIP 顶层目录必须为 `Empire Earth II/`
2. `launcher` 包 ZIP 顶层目录必须为 `地球帝国二代远航版启动器/`
3. 启动器只负责检查 `latest.json`、展示更新提示并拉起独立更新器。
   - 若 `latest.json` 未提供 `packages.launcher`，视为本次不需要启动器自升级。
4. 独立更新器在 `scope=all` 模式下按 `launcher -> game` 顺序应用补丁。
5. `game` 包版本未完成时，客户端必须进入阻塞式强更流程。
6. 只允许覆盖 manifest 中声明的文件。
7. 只允许删除 `deleteList` 中声明的路径。
8. `protectedPaths` 永远跳过，不允许被发布包覆盖。

补充：

- 本协议采用“补丁模式”而不是“最终完整文件清单模式”。
- `files[]` 仅表示“本次要新增/覆盖的文件集合”。
- 未出现在 `files[]` 的旧文件保持原状，不能被自动推断为删除。
- 第一版默认禁用自动删除推断；若未来需要删除文件，必须通过显式 `deleteList` 表达。

## release-state.json

新版 `update/runtime/release-state.json` 采用嵌套结构：

```json
{
  "launcher": {
    "version": "1.2.3",
    "packageSha256": "..."
  },
  "game": {
    "version": "1.2.3",
    "packageSha256": "..."
  },
  "pendingVersion": "",
  "releaseNotes": "...",
  "lastShownChangelogVersion": "1.2.3",
  "appliedAt": "2026-05-20T00:00:00Z",
  "version": "1.2.3",
  "packageSha256": "..."
}
```

兼容说明：

- `version` 与 `packageSha256` 仍保留为旧字段，等价于 `game` 包当前状态。
- 只有当 `game.version == latest.version` 时，客户端才应把该版本视为“最终已完成”。
- 首次安装目录缺少 `release-state.json` 时，客户端允许基于 `update/version_history.json` 与 `package.json` 自动补写本地 bootstrap 状态。

## Flutter 发布端行为

- 版本号与更新内容在主界面右侧常驻编辑，不再通过推送弹窗录入。
- 删除清单不再由人工输入。
- Flutter 只负责本地构建双包，并将整个 `releaseDir` 导出为单个更新包 ZIP。
- 默认先保存到打包工具同级 `data/zip/` 目录。
- 默认导出文件名：`ee2x-{version}-{YYYYMMDD-HHmmss}.zip`
- 导出完成后可通过“另存到...”按钮复制到自定义位置。
- 外层 ZIP 根目录固定包含：
  - `release-meta.json`
  - `release-notes.txt`
  - `launcher/release-manifest.json`
  - `launcher/EE2X-launcher-{releaseId}.zip`
  - `game/release-manifest.json`
  - `game/EE2X-game-{releaseId}.zip`
- `release-meta.json` 额外包含：
  - `bundleFormat = "ee2x-release-bundle"`
  - `bundleVersion = 1`
- 第一版 `deleteList` 默认由客户端保持为空，服务端不再根据“上一版有、这次没选”自动推断删除。
- 若后续开放显式删除功能，服务端只校验并原样保存客户端 manifest 中带来的 `deleteList`。

## Web 推送页行为

- Web 推送页挂载到现有 `3010` 后端：
  - `GET /publish/`
- 静态资源路径：
  - `GET /publish/assets/*`
- 页面登录方式：
  - 账号密码
  - 默认 `ee2x / ee2x`
- 页面只上传单个 bundle ZIP，不再由桌面端直连发布。
- 首版固定发布到服务端默认频道，不暴露 channel 切换。
- 上传接口：
- `POST /api/update/v1/releases/publish-bundle`
- `GET /api/update/v1/channels/{channel}/history?limit=0`

补充：

- `history` 在 `limit=0` 时返回该频道全部已发布版本。
- 每个历史条目除原有版本号、发布时间、更新说明外，还会返回：
  - `downloadCount`（按游戏包统计的版本下载量）
  - `launcherDownloadCount` / `gameDownloadCount`
  - `launcherPackageSize` / `gamePackageSize`
  - `lastDownloadedAt`
- `/updates/{channel}/releases/{releaseId}/{scope}/{zipFile}` 由后端显式提供，以便在实际下载时自动累计版本下载次数。
