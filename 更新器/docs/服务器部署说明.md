## 新后端接口

### API

- `GET /api/update/v1/health`
- `GET /api/update/v1/channels/{channel}/latest`
- `GET /api/update/v1/channels/{channel}/history?limit=0`
- `POST /api/update/v1/releases/publish`
- `POST /api/update/v1/releases/publish-bundle`
- `DELETE /api/update/v1/channels/{channel}/releases/{releaseId}`

### Web 推送页

- `GET /publish/`
- `GET /publish/assets/*`

### 静态下载

- `GET /updates/{channel}/latest.json`
- `GET /updates/{channel}/releases/{releaseId}/{scope}/release-manifest.json`
- `GET /updates/{channel}/releases/{releaseId}/{scope}/{zipFile}`

### 兼容接口

- `GET /manifest`
- `GET /api/version/latest`
- `GET /api/version/history`

说明：

- 启动器新流程优先使用 `/api/update/v1/...`
- 外部更新器继续使用 `/updates/...`
- 兼容接口用于兜住历史代码路径，避免残余逻辑直接报错
- `history` 返回中包含 `currentReleaseId` / `currentVersion`，供发布端标记当前 latest
- `history` 在 `limit=0` 时返回该频道全部历史版本，并附带下载量、包体大小、最后下载时间等详情字段
- `/updates/.../{zipFile}` 下载现在由后端显式处理，用于对每个版本的 launcher/game 包累计下载次数

## 发布配置

本地文件：

- `更新器/config/publish.local.json`

字段：

```json
{
  "backendBaseUrl": "http://115.231.35.105:3010",
  "adminUsername": "ee2x",
  "adminPassword": "ee2x",
  "channel": "stable"
}
```

说明：

- 不再使用：
  - `sshHost`
  - `sshPort`
  - `sshUsername`
  - `sshPassword`
  - `sshPrivateKey`
  - `remoteRoot`
  - `publicBaseUrl`
- Web 推送页与兼容 API 现在使用账号密码 Basic Auth。
- 默认账号密码：
  - `ee2x / ee2x`
- 桌面端现在不再依赖这份配置执行打包；它只保留给 Web/API 发布链路与兼容 CLI 使用。

## 部署与维护

### 首次部署

本地脚本：

- `更新器/update_backend_mg/deploy_backend.py`

用途：

- 上传后端源码
- 先备份当前远端 `db/`、`storage/`、`.env` 与 systemd 服务文件
- 创建远端 venv
- 安装依赖
- 保留现有 `.env`（首次部署才写入新 `.env`）
- 注册 `systemd`
- 仅在首次部署且远端无现有数据时才导入旧更新目录
- 重启并拉起 `3010`

当前脚本的默认备份目录：

- `/root/ee2x/backups/ee2x-update-mg-YYYYMMDD-HHmmss/`

### 重建索引

远端命令：

```bash
cd /root/ee2x/ee2x_up-mg
./.venv/bin/python -m app.import_legacy --rebuild-only
systemctl restart ee2x-update-mg.service
```

### 服务状态

```bash
systemctl status ee2x-update-mg.service
ss -ltnp | grep 3010
```
