# 单位属性文档模板

> 本模板用于提取任意文明、任意时代的单位属性。新建文档时复制此模板并填充数据。
> 模板版本: v1.0 | 创建日期: 2026-05-18

---

## 一、文档命名规范

| 规则 | 格式 | 示例 |
|:-----|:-----|:-----|
| 目录 | `单位属性文档\{文明}\` | `单位属性文档\俄罗斯\` |
| 文件名 | `{文明}_{时代}_单位属性.md` | `俄罗斯_时代14_单位属性.md` |
| 分类文件 | `{文明}_{时代}_{兵种}_单位属性.md` | `中国_时代14_海军单位属性.md` |

---

## 二、文档头部模板

```markdown
# {文明名}文明 — 第{时代}时代 {兵种}单位属性总表

> 数据来源：dbtechtreenode.csv + upgrade_unittypes.csv + upgrade_unittypes_EE2X.csv + DDF文件

> 数据文件路径：game-metadata/EE2X_db/
> 关联快照：进度快照\{序号}-{描述}.md
```

### 4.2 DDF-Only单位（无CSV升级条目）

与标准单位相同，但：
- DDF完整参数表中各项标注为基础值（无CSV覆盖）
- 数据来源位置表中省略"CSV E{时代}升级"行
- 添加说明：`> 无CSV升级条目，属性全由DDF定义。`

```markdown
**数据来源位置**:

| 数据类型 | 文件路径 | 行号 |
|:---------|:--------|:-----|
| DDF属性 | Units/{文件名}.ddf | {行号} |
| 科技树节点 | TechTree/dbtechtreenode.csv | {行号} |
```

### 4.3 注释/废弃单位

在基本信息表中增加 `| 状态 | CSV行注释 (//) + 科技树节点注释 |` 行，并在末尾说明替代关系：

```markdown
| 状态 | CSV行注释 (//) |

> 已被 {替代单位} 替代。
```

## 八、数据提取工作流

按以下顺序提取每个单位的数据：

```
1. 搜索CSV → grep "{UnitType}UpgradeEpoch{时代}" upgrade_unittypes.csv
               记录行号和CSV值（HP/LOS/DAMAGE/RANGE/RELOAD/BUILDTIME/各资源造价）

2. 搜索DDF → grep "UnitType {UnitType}" Units/*.ddf
               定位UnitType块起止行号
               逐行读取 properties / abilities / attributes 块
               提取所有非零参数

3. 搜索科技树 → grep "^{UnitType}," dbtechtreenode.csv
                  记录行号、建造资源、建造时间

4. 搜索升级集 → grep "{UpgradeName}" epoch{时代}_upgrades.ddf
                  记录行号（可能有多个引用位置）

5. 核对一致性:
   - DDF基础值 vs CSV E{时代}覆盖值 → 标注在DDF参数表的"说明"列
   - 若CSV值与DDF值差异大，标注为"CSV覆盖"
   - 若某参数仅在DDF中存在，标注为"DDF基础值"
```

---

## 九、格式铁律

1. **每个单位必须有 数据来源位置 表** — 精确到文件名+行号
2. **DDF参数必须中文标注+英文原名** — 如 `生命值 (HitPoints)`
3. **CSV覆盖关系必须标注** — DDF参数表"说明"列标注"基础值，E14 CSV覆盖为{值}"
4. **科技树造价必须列出** — 每个单位末尾的独立行
5. **DDF-only单位必须标注** — "无CSV升级条目，属性全由DDF定义"
6. **注释/废弃单位必须标注状态** — 基本信息表包含"状态"行
7. **通用单位表必须包含全资源造价列** — 食物/木材/石头/黄金/石油/铀
8. **数据来源说明表必须列出所有引用的DDF文件**
