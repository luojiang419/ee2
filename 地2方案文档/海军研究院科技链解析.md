# NavalAcademy（海军研究院）科技项目设置分析

## 1. 整体逻辑

- **建筑本身**：在 `dbtechtreenode.csv` 里有一条 **NAME=NavalAcademy** 的节点，**HOST=Citizen**，**PRODUCE=NavalAcademy**，表示在城镇中心（Citizen）里 9 时代可造海军研究院。
- **科技项**：所有 **HOST=NavalAcademy** 的节点，都会出现在「海军研究院」建筑的科技面板里；每个节点的 **UPGRADE** 指向一个 **UpgradeRefSet**（在对应时代的 `epoch*_upgrades.ddf` 里定义），该 RefSet 里列出的才是实际生效的单位升级 id。

即：**谁出现在海军研究院里**由 CSV 的 HOST 决定；**点了之后解锁哪些单位升级**由 DDF 里同名 UpgradeRefSet 决定。

---

## 2. 数据来源

| 用途 | 文件 | 作用 |
|------|------|------|
| 建筑解锁 + 科技项列表与成本 | `EE2X_db\TechTree\dbtechtreenode.csv` | 定义 NavalAcademy 建筑节点，以及 HOST=NavalAcademy 的科技节点（NAME、UPGRADE、EPOCH、成本、PREREQUISITES 等） |
| 科技项实际效果（解锁哪些单位升级） | `EE2X_db\TechTree\epoch11_upgrades.ddf` ~ `epoch15_upgrades.ddf` | 每个科技节点的 UPGRADE（如 upFregattenSlot1_12）对应一个 **UpgradeRefSet**，其 `upgrades = [ ... ]` 列出要解锁的 upgrade id |
| 建筑单位类型与模型 | `EE2X_db\Units\Tao_Academy.ddf` | 定义 UnitType NavalAcademy（血量、能力等）和 UnitModel |
| 领土限制 | `EE2X_db\Simulation\Misc\territory.ddf` | 每领土 NavalAcademy count = 1 |

## 3. dbtechtreenode.csv 中的设置

### 3.1 海军研究院建筑节点（一条）

```text
NAME          = NavalAcademy
HOST          = Citizen
PRODUCE       = NavalAcademy
UPGRADE       = （空）
EPOCH         = 9
TIME/资源等   = 60, 0, 800, 500, 700, ...
```

表示：9 时代在城镇中心可建造「海军研究院」，建造后该建筑作为 HOST，下面所有 HOST=NavalAcademy 的科技节点会显示在它的科技面板里。

### 3.2 科技节点（HOST=NavalAcademy）

每个科技项一行，主要字段含义：

- **NAME**：科技节点 id（界面/逻辑用）。
- **HOST**：= NavalAcademy → 显示在海军研究院里。
- **PRODUCE**：空（这里不是生产单位，是研发科技）。
- **UPGRADE**：对应 DDF 里的 **UpgradeRefSet 名**（如 upFregattenSlot1_11、upDestroyerSlot1_12）。
- **EPOCH**：该科技可研发的时代。
- **SLOT**：= 4（Improvement/Military 槽）。
- **BRANCH**：Military 或 Improvement，用于分类/图标。
- **ROW / COLUMN**：面板上的行列位置（0, 1, 2…）。
- **TIME, FOOD, WOOD, STONE, GOLD, …**：研发所需时间和资源。
- **TECHPTS**：科技点。
- **PREREQUISITES**：前置科技（如 upDestroyer_12 依赖 [upFregatten_12]）。

当前配置摘要：

| NAME | EPOCH | COLUMN | UPGRADE | 前置 PREREQUISITES |
|------|-------|--------|---------|--------------------|
| upFregatten_11 | 11 | 0 | upFregattenSlot1_11 | — |
| upFregatten_12～15 | 12～15 | 0 | upFregattenSlot1_12～15 | — |
| upDestroyer_12～15 | 12～15 | 1 | upDestroyerSlot1_12～15 | upFregatten_同时代 |
| upBattleship_12～15 | 12～15 | 2 | upBattleshipSlot1_12～15 | upDestroyer_同时代 |
| upVLSTech_14 | 14 | 2 | upVLSTechSlot1_14 | upBattleship_14 |

同一列（COLUMN）表示同一「科技链」的一档（护卫舰→驱逐舰→战列舰→VLS）。

## 4. epoch*_upgrades.ddf 中的设置（实际效果）

每个 **UPGRADE**（如 upFregattenSlot1_12）在对应时代的 DDF 里有一个 **UpgradeRefSet**：

```ddf
UpgradeRefSet upFregattenSlot1_12 {
	upgrades = [
		Ch054AUpgradeEpoch12
		FregattenFFGUpgradeEpoch12
		Type22350UpgradeEpoch12
		…  // 更多单位升级 id
	]
}
```

- **upgrades** 列表中的每一个 id，都必须在 `upgrade_unittypes.csv` 或 `upgrade_unittypes_EE2X.csv` 里存在对应行（第一列为该 id）。
- 玩家在海军研究院里研发该科技后，这些单位升级会被解锁（可生产对应时代/型号的舰船）。

不同 Slot 大致对应：

- **upFregattenSlot1_***：护卫舰/潜艇等一档舰船升级。
- **upDestroyerSlot1_***：驱逐舰/巡洋舰等二档。
- **upBattleshipSlot1_***：战列舰等三档。
- **upVLSTechSlot1_14**：VLS 相关单位（仅 14 时代）。

注意：**epoch11_upgrades.ddf** 里 `upFregattenSlot1_11` 的 `upgrades = [ ]` 为空，即 11 时代护卫舰线在海军研究院里没有解锁任何单位，仅占位；实际可解锁内容需在 DDF 中补写 upgrade id。

## 5. 如何修改「海军研究院里的科技」

1. **增/删/改「科技项」**  
   - 在 **dbtechtreenode.csv** 中增删或修改 **HOST=NavalAcademy** 的行（NAME、UPGRADE、EPOCH、ROW/COLUMN、成本、TECHPTS、PREREQUISITES）。

2. **改某个科技解锁的单位**  
   - 在对应 **epoch*_upgrades.ddf** 里找到同名 **UpgradeRefSet**（与 CSV 的 UPGRADE 一致），修改其 `upgrades = [ ... ]` 列表（单位升级 id 必须存在于 upgrade_unittypes 或 upgrade_unittypes_EE2X 中）。

3. **改建筑本身**  
   - 建筑节点：改 dbtechtreenode 里 NAME=NavalAcademy 那一行（时代、成本等）。  
   - 建筑属性/模型：改 Tao_Academy.ddf 里 UnitType NavalAcademy 和 UnitModel。

---

## 6. 小结

- **谁在海军研究院里、花多少资源/科技点、依赖谁**：由 **dbtechtreenode.csv** 中 **HOST=NavalAcademy** 的节点和其 **UPGRADE**、**PREREQUISITES** 决定。
- **每个科技具体解锁哪些舰船**：由 **epoch*_upgrades.ddf** 里与 UPGRADE 同名的 **UpgradeRefSet** 的 `upgrades = [ ... ]` 决定。
- 二者通过 **UPGRADE 名 = UpgradeRefSet 名** 一一对应（例如 upFregattenSlot1_12 既在 CSV 的 UPGRADE 列出现，也在 epoch12_upgrades.ddf 里作为 UpgradeRefSet 名出现）。
