# dbtechtreenode.csv 与升级定义核对报告

## 检查方法

- 从 `EE2X_db\TechTree\dbtechtreenode.csv` 提取所有非空的 **UPGRADE** 引用。
- 与 `upgrade_unittypes.csv`、`upgrade_unittypes_EE2X.csv` 的**第一列（UPGRADE id）**比对（仅统计**未注释**行）。

## 1. 单位升级：定义被注释（科技树仍引用）

以下 UPGRADE 在科技树节点中被引用，但在 `upgrade_unittypes.csv` 中**整行被注释**（`//`），游戏不会加载这些行，因此等于「没有定义」：

| UPGRADE | 科技树节点 (NAME) | HOST | 说明 |
|---------|-------------------|------|------|
| **SiegeshipUpgradeEpoch3** | Siegeship | Dock | 攻城船 Epoch3 升级行被注释 |
| **Destroyer055UpgradeEpoch11** | Destroyer055 | Dock | 中国驱逐舰 055 E11 行被注释 |
| **RusskirovUpgradeEpoch11** | Russkirov | Dock | 俄国巡洋舰 E11 行被注释 |
| **Ticonderoga14UpgradeEpoch11** | Ticonderoga14 | Dock | 美国提康德罗加 E11 行被注释 |
| **UU_Japanese3UpgradeEpoch11** | UniqueUnitJapanese3 | Zuikaku | 日本 UU3 E11 行被注释（E12–E15 未注释） |
| **J11DUpgradeEpoch13** | J11D | Airport | 中国 J11D E13 行被注释 |
| **IranAH1UpgradeEpoch13** | IranAH1 | Airport2 | 伊朗 AH1 E13 行被注释 |
| **TurkeyAH1UpgradeEpoch13** | TurkeyAH1 | Airport2 | 土耳其 AH1 E13 行被注释 |

**建议**：若希望这些单位在科技树中可用，请在 `upgrade_unittypes.csv` 中**取消对应行的注释**（删除行首 `//`）。若为刻意禁用，可在科技树中注释或删除对应节点。

---

## 2. Brewery（酿酒厂）

- 科技树节点：`Brewery`（NAME），UPGRADE = `BreweryUpgradeEpoch6`；该节点在 `dbtechtreenode.csv` 中**已被注释**（`//Brewery`）。
- `upgrade_unittypes*.csv` 中未找到 `BreweryUpgradeEpoch6`（无论是否注释）。
- 若之后取消注释 Brewery 节点，需在升级表中增加或取消 `BreweryUpgradeEpoch6` 的对应行。

---

## 3. Elite / Veteran 与 UniqueUnit*Elite|Veteran

大量「缺失」引用形如：

- `HeavyInfantry1Epoch1Elite` / `HeavyInfantry1Epoch1Veteran`
- `BattleshipEpoch11Elite` / `BattleshipEpoch11Veteran`
- `UniqueUnitFrench1Elite` / `UniqueUnitFrench1Veteran`
- 等

这些多为**老兵/精英状态**的引用，可能由引擎或 DDF/脚本通过其他方式绑定，而不在 `upgrade_unittypes` 里占一行。若游戏内这些单位升级/精英显示正常，可视为预期，无需在 unittypes 表里补行。

---

## 4. 其他 679 条未匹配引用

除上述单位相关项外，其余未匹配多为：

- **EconomicEpoch*Slot***、**ImperialEpoch*Slot***、**MilitaryEpoch*Slot***：可能来自 `upgrade_factorset.csv` 或 DDF。
- **Traits_***：因子类加成，多在 factorset 或别处定义。
- **up*Slot***（如 upBattleshipSlot1_12）：槽位/逻辑用，不一定在 unittypes 表。

若需严格一致，可再对 `upgrade_factorset.csv` 与各 `epoch*_upgrades.ddf` 做一次 id 收集，把「仅出现在 factorset/DDF」的从「缺失」中排除。

---

## 小结

- **需要优先处理**：8 个单位升级在科技树中被引用，但定义在 `upgrade_unittypes.csv` 里被注释，若要在游戏中使用这些单位，请取消对应行注释。
- **Brewery**：节点已注释；若将来启用，需补 `BreweryUpgradeEpoch6` 定义。
- **Elite/Veteran 与 Traits/Economic 等**：多数为它处定义或引擎逻辑，可按实际表现决定是否在 unittypes 中补定义。
