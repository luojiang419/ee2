# EE2X 数据查找与升级分析参考

本文档概述在 Empire Earth 2 Expansion 数据库中
查找单位、升级和按钮位置的标准流程。遵循此逻辑
可长期复用，保证分析一致性。

---

## 1. 单位基础信息

1. 打开 `TechTree\dbtechtreenode.csv`。
2. 搜索目标单位名称（例如 `Lumber`、`Ch055`）。
3. 读取该行数据：
   - `Epoch` 与 `AvailableEpoch`：可用纪元
   - `Upgrades` 字段：可能包含 `UpgradeEpochX` 或升级节点
   - `Menu/Row/Col`：按钮位置
   - `ObjectName`、`RPS`、`Cost` 等基础属性
   - `Civilization` 字段：如果为空则为通用单位，非空为特色单位

> 注意：有些行仅用于 AI 或被 UI 隐藏，后续步骤会处理。

## 2. 升级类型判断

- **自动升级**：单位的 `UpgradeEpochX` 被加入主纪元集合
  (`MainEpochX` 等)或者其他包含了 `AdvanceEpochX` 的 UpgradeRefSet。
  玩家进入该纪元后自动替换属性。

- **主动升级**：单位对应一个升级节点（例如
  `upBattleshipSlot1_12`、`upVLSTech_14`），且在 dbtechtreenode.csv
  指定了按钮位置。玩家必须在相应建筑中点击才能生效。

- **AI 专用/隐藏**：按钮位置超出 UI 范围（col >3）、或被其他单位
  覆盖、或属于 `AICoastalDefense` 等特殊节点，则无法显示给玩家。

## 3. 查找升级属性

1. 若存在升级节点或 `UpgradeEpochX`，转到
   `TechTree\upgrade_unittypes.csv`。
2. 使用节点名检索（例如 `Ch055UpgradeEpoch14`、
   `T99AUpgradeEpoch14`）。
3. 读取升级行：
   - HP、Damage、Range、Reload、LOS
   - 资源成本（Food/Wood/Gold/...）
   - 外观字段 `ObjectName`（如 `Ch055_14`）
   - 建造时间、Speed 等
4. 如果找不到条目，则说明单位在当前纪元没有额外升级，使用
   dbtechtreenode.csv 的属性。

> 切记：尽管名称可能混乱（`Ch055Epoch13Attack` 用于第14纪元），
> 只要引用正确的 `UpgradeAbilities` 定义即可。

## 4. 按钮位置与UI可见性

- `menu`、`row`、`col` 决定按钮在建筑界面的位置。UI 支持最多3列，
  超出范围的按钮为隐藏。
- 若两个单位位置一致，后出现的单位覆盖前者。
- 结合文明筛选（只显示本文明的通用/特色单位）。

## 5. 丰富单位信息

- 如果需要模型信息，查 `Units/*.ddf`：
  - `ForceScale` 决定实体大小
  - `ChildNames` 是特效集合
  - `UIRenderingParams` 仅用于 UI 框，不影响模型

- 范围伤害：在攻击能力中查看 `areaDamageRadius` 和 `areaDamageFriendlyRatio`。

- 额外能力如 `NavalMove`、`Attack` 参数是升级效果的重要组成。

## 6. 特例提醒

- `AICoastalDefense` 等单位按钮位置不可见，仅 AI 可造。
- `upVLSTech_14` 等升级集合代表特定技术（VLS）而非单一单位升级。
- 通用单位无文明字段；特色单位必须按文明归属查询。

## 7. 自动化建议

