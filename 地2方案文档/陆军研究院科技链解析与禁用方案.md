# ArmyAcademy（陆军研究院）工作原理分析与禁用方案

## 1. 整体逻辑

陆军研究院（ArmyAcademy）是远航版在时代9新增的军事建筑，功能是为玩家提供**陆军单位升级科技**的解锁入口。

核心工作原理分三层：

1. **建筑本身**：`dbtechtreenode.csv` 中 NAME=ArmyAcademy 节点，HOST=Citizen，PRODUCE=ArmyAcademy → 市民（Citizen）在时代9可建造此建筑
2. **科技面板**：CSV 中所有 HOST=ArmyAcademy 的节点会出现在该建筑的科技面板中
3. **实际效果**：每个科技节点的 UPGRADE 字段指向 `epoch*_upgrades.ddf` 中的 UpgradeRefSet，RefSet 的 `upgrades = [...]` 列出实际解锁的单位升级ID

即：**谁出现在陆军研究院里**由 CSV 的 HOST 决定；**点了之后解锁哪些单位升级**由 DDF 里同名 UpgradeRefSet 决定。

## 3. 建筑定义（Tao_Academy.ddf 第1-68行）

```
UnitType ArmyAcademy {
    parent = Building
    properties {
        HitPoints = 10000
        displayName = tx_utn_ArmyAcademy_name
    }
    abilities = [
        LOS { range = 5 }
    ]
    icon = icon_unit_ArmyAcademy
}

UnitModel ArmyAcademy_9  → ArmyAcademy10.NIF  (时代9-11外观)
UnitModel ArmyAcademy_12 → ArmyAcademy13.NIF  (时代12+外观)
```

- parent = Building → 继承 NoProduceBuilding → 拥有 `Research {}` 能力（可研发科技）
- abilities 仅 LOS（视野），无驻军、无生产
- 有两个时代的模型外观

## 4. 建筑解锁节点（dbtechtreenode.csv 第21行）

```csv
NAME=ArmyAcademy, HOST=Citizen, EPOCH=9, PRODUCE=ArmyAcademy, UPGRADE=ArmyAcademyUpgradeEpoch9
TIME=60, FOOD=500, WOOD=500, STONE=500
```

含义：市民在时代9可花费 500食物+500木材+500石料，用60秒建造陆军研究院。

|-----------|----------|------|-----|----------|
| ArmyAcademyUpgradeEpoch9 | ArmyAcademy | 9 | 9000 | ArmyAcademy_9 |
| ArmyAcademyUpgradeEpoch12 | ArmyAcademy | 13 | 10000 | ArmyAcademy_9 |
| ArmyAcademyUpgradeEpoch15 | ArmyAcademy | 15 | 13000 | ArmyAcademy_9 |

## 6. epoch*_upgrades.ddf 中的 UpgradeRefSet

每个科技节点的 UPGRADE 字段对应一个 UpgradeRefSet，位于对应时代的 DDF 文件中：

| UpgradeRefSet | 所在文件 | 含义 |
|---------------|----------|------|
| upArmoredCarSlot1_11 | epoch11_upgrades.ddf | 装甲车时代11升级 |
| upArmoredCarSlot1_12 ~ _15 | epoch12~15_upgrades.ddf | 装甲车时代12~15升级 |
| upLightTnakSlot1_12 ~ _15 | epoch12~15_upgrades.ddf | 轻型坦克升级 |
| upHeavyTankSlot1_12 ~ _15 | epoch12~15_upgrades.ddf | 重型坦克升级 |
| upAirDefenseSlot1_13 ~ _15 | epoch13~15_upgrades.ddf | 防空升级 |
| upLightArtillerySlot1_12 ~ _15 | epoch12~15_upgrades.ddf | 轻型火炮升级 |
| upHeavyArtillerySlot1_12 ~ _15 | epoch12~15_upgrades.ddf | 重型火炮升级 |
| upJ16Slot1_13 | epoch13_upgrades.ddf | J16战机升级 |
| upJ16Slot1_14 | epoch14_upgrades.ddf | J16时代14升级 |
| upMissloSlot1_14 | epoch14_upgrades.ddf | 导弹升级 |
| upNuclearSlot1_14 | epoch14_upgrades.ddf | 核弹（已定义但节点已注释） |

每个 UpgradeRefSet 的 `upgrades = [...]` 中列出具体的单位升级ID（如 HeavyTankUpgradeEpoch12 等），这些ID必须在 `upgrade_unittypes.csv` 或 `upgrade_unittypes_EE2X.csv` 中存在对应定义。

## 7. 禁用方案

### 方案A：仅禁用科技面板（保留建筑）

**操作**：注释 `dbtechtreenode.csv` 中所有 HOST=ArmyAcademy 的科技节点行（第40-68, 81, 90-91行），在行首加 `//`

**效果**：建筑仍可建造，但科技面板为空，无法研发任何陆军科技。

**涉及行**（共24行）：
```
//upArmoredCar_11,text_upArmoredCar_11_name,...ArmyAcademy...
//upArmoredCar_12,...
...（所有 HOST=ArmyAcademy 的科技节点）
```

**优点**：可逆性强，改动最小
**缺点**：建筑本身仍在游戏中，玩家可以建造但无功能

---

### 方案B：完全禁用建筑（推荐）

**操作**：注释 `dbtechtreenode.csv` 中以下内容：
1. 建筑解锁节点（第21行）：`//ArmyAcademy,...`
2. 所有 HOST=ArmyAcademy 的科技节点（第40-68, 81, 90-91行）

**效果**：市民无法建造陆军研究院，建筑完全从游戏中移除。

**同时建议**：注释以下升级引用，避免时代推进时出现孤立升级：
3. `epoch12_upgrades.ddf` 第33行：`//ArmyAcademyUpgradeEpoch12`
4. `epoch15_upgrades.ddf` 第422行：`//ArmyAcademyUpgradeEpoch15`

**注意**：不需要注释 `upgrade_unittypes.csv` 中的建筑升级定义，因为它们仅在建筑存在时才被引用。

---

### 方案C：仅禁用部分科技链

**操作**：只注释特定科技链对应的行。

例如只保留坦克线，注释掉火炮线和防空线：

| 保留 | 注释的节点 |
|------|-----------|
| 装甲车线 | — |
| 轻型坦克线 | — |
| 重型坦克线 | — |
| 防空线 | upAirDefense_13~15 |
| 轻型火炮线 | upLightArtillery_12~15 |
| 重型火炮线 | upHeavyArtillery_12~15 |
| J16 | upJ16_13 |
| 导弹 | upMisslo_14 |

**优点**：灵活定制
**缺点**：需要注意依赖链，被注释节点的后续节点也会失效

---

### 方案D：修改建筑为装饰（保留外观）

**操作**：
1. 执行方案A（注释所有科技节点）
2. 在 `Tao_Academy.ddf` 中修改 ArmyAcademy 的 abilities，移除 Research 能力（如果通过父类继承的）

**效果**：建筑可作为地图装饰或占领目标，但无实际科技功能。

---

## 8. 推荐方案

**推荐方案B（完全禁用建筑）**，操作步骤：

1. **备份文件**
   - dbtechtreenode.csv
   - epoch12_upgrades.ddf
   - epoch15_upgrades.ddf

2. **修改 dbtechtreenode.csv**（注释以下行）：
   - 第21行：建筑解锁节点 → `//ArmyAcademy,text_CityCenter_name,...`
   - 第40-44行：装甲车线（upArmoredCar_11~15）
   - 第46-49行：轻型坦克线（upLightTnak_12~15）
   - 第51-54行：重型坦克线（upHeavyTank_12~15）
   - 第56-58行：防空线（upAirDefense_13~15）
   - 第60-63行：轻型火炮线（upLightArtillery_12~15）
   - 第65-68行：重型火炮线（upHeavyArtillery_12~15）
   - 第81行：J16（upJ16_13）
   - 第90行：导弹（upMisslo_14）
   - 第91行：核弹（已注释，保持不变）

3. **修改 epoch12_upgrades.ddf** 第33行：
   ```
   //ArmyAcademyUpgradeEpoch12
   ```

4. **修改 epoch15_upgrades.ddf** 第422行：
   ```
   //ArmyAcademyUpgradeEpoch15
   ```

5. **测试验证**：启动游戏，进入时代9+的存档，确认：
   - 市民建造菜单中无"陆军研究院"
   - 已建造的陆军研究院（旧存档）不影响游戏运行
   - 时代推进无报错

## 9. 注意事项

1. **已建造的建筑**：如果玩家已有存档中建造了陆军研究院，禁用建筑节点不会自动移除已建造的建筑，但科技面板会变为空。
2. **关联单位**：陆军研究院解锁的是单位**升级**而非单位本身。禁用研究院后，对应的基础单位（如基础坦克、火炮）仍可通过兵营等建筑生产，只是无法通过研究院的科技升级到更高时代的型号。
3. **AI影响**：AI 的建筑策略中可能包含建造陆军研究院的逻辑，禁用后 AI 会跳过此建筑。
4. **建筑升级引用**：ArmyAcademyUpgradeEpoch12/15 在 MainEpoch12/MainEpoch15 的 UpgradeRefSet 中，这些升级集是时代推进时批量执行的，注释后不会影响其他升级。
