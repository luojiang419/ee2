### 关键文件格式说明

| 格式 | 说明 | 编辑工具 |
|:-----|:-----|:---------|
| `.csv` | 逗号分隔数据表（首行列名，`//` 开头为注释行） | Excel / VS Code / 任意文本编辑器 |
| `.ddf` | 自定义键值对/块格式（单位/建筑/属性定义） | 文本编辑器 |
| `.utf8` | UTF-8编码的文本字符串 | 文本编辑器 / Poedit |
| `.h` | C/C++头文件（枚举/结构体定义） | 文本编辑器 |
| `.zip` | 压缩资源包（解压后修改，重新打包或放置同名文件夹覆盖） | 解压工具 + 重打包 |
| `.ies` | Empire Earth 源脚本 | 文本编辑器 |
| `.aip` | AI人格脚本 | 文本编辑器 |
| `.cfg` | 游戏配置文件 | 文本编辑器 |
| `.scn` | 场景/战役文件 | 编辑器或16进制工具 |

### 数据修改标准流程

根据前任开发者文档《数据查找标准流程指南.md》：

1. **定位目标单位** → 打开 `dbtechtreenode.csv`，搜索单位名称
2. **读取属性** → 查看 Epoch / Upgrade / Menu-Row-Col / ObjectName / RPS / Cost / Civilization
3. **确认升级方式** → 判断是自动升级(UpgradeEpochX)还是手动升级(按钮Column ≤ 3)
4. **查找升级数据** → 在 `upgrade_unittypes.csv` 中搜索对应升级节点名
5. **检查模型/能力** → 在 `Units/*.ddf` 中查看ForceScale、ChildNames、areaDamage、popCount等
6. **验证关联** → 确保 `dbtechtreenode.csv` 的 UPGRADE 列名称能在 `upgrade_unittypes.csv` 中找到
7. **同步UI文本** → 同步更新 `Text/` 目录下的 `.utf8` 文件

## 八、重要注意事项

### ⚠️ 修改陷阱（来自前任开发者经验）

1. **CSV不是唯一数据源**：DDF文件中的 `UpgradeAbilities` 和 `UpgradeSize` 块会覆盖CSV中的数值
2. **人口容量**：CSV中的人口列对船只无效，实际入口消耗在DDF的 `popCount` 字段
3. **RPS类型**：单位的石头-剪刀-布类型由DDF的 `rps` 字段决定，不是CSV或中文名称
4. **升级引用一致性**：`dbtechtreenode.csv` 的 UPGRADE 列值必须能在 `upgrade_unittypes.csv` 中找到对应行，否则升级静默失效
5. **特殊资源合法性**：修改升级成本时需验证资源是否在该时代可用
   - Tin(锡): E1-E6, Iron(铁): E4-E9, Saltpeter(硝石): E7-E12, Oil(石油): E10-E15, Uranium(铀): E13-E15
6. **行号依赖**：`upgrade_unittypes.csv` 修改行数后，如果外部工具按行号引用（而非按名称），会导致引用错误
7. **EXE硬编码**：某些常数（文明属性列表、游戏速度倍率等）在 `EE2X.exe` 中硬编码，仅修改数据文件无法完全生效
8. **文本同步**：修改枚举/选项数量时，需同步更新 `DbFrontEndDefs.h` 和所有 `dbtext_*.utf8` 文件
