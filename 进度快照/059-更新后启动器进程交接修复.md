# 进度快照 059 — 更新后启动器进程交接修复

> 日期: 2026-05-22

---

## 已完成内容

### 1. 已确认问题更像“旧进程未完全退出 + 单实例锁折返”

- 现象:
  - 由更新器拉起的启动器窗口几乎瞬间出现
  - 此窗口内点击“开始游戏”会报 `UP15.dll` 找不到
  - 手动彻底关闭后，再手动启动启动器，又能正常进游戏
- 结论:
  - 更像是新实例启动时被旧实例的 `requestSingleInstanceLock()` 拦截
  - `second-instance` 事件把焦点又交回了旧窗口

### 2. 启动器已新增“进程替换准备”逻辑

- 修改文件:
  - `Empire Earth II/地球帝国二代远航版启动器/Core/resources/app/main.js`
- 新增:
  - `prepareForLauncherReplacement(reason)`
- 行为:
  - 标记 `launcherReplacementInProgress = true`
  - 设置 `app.isQuitting = true`
  - 主动释放单实例锁
  - 隐藏并销毁旧窗口
  - 1.5 秒后若旧进程仍未退出，则执行强制退出兜底

### 3. second-instance 折返已被拦截

- 旧逻辑:
  - 新实例拿不到锁时，会触发旧实例 `second-instance`
  - 旧实例会把自己窗口重新显示并聚焦
- 新逻辑:
  - 如果旧实例已处于 `app.isQuitting` 或 `launcherReplacementInProgress`
  - 则忽略 `second-instance`
  - 不再把用户重新送回旧窗口

### 4. 已接入两条重启路径

- `launcher:runUpdater` → `quitLauncherForUpdater`
  - 更新器接管时会先释放锁，再退出旧启动器
- `launcher:restart`
  - 手动重启时也走同一套进程替换逻辑

### 5. 已完成语法校验

- 使用:
  - `C:\Program Files\nodejs\node.exe --check`
- 校验结果:
  - `main.js` 语法通过
