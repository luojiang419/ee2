# 进度快照 083 — 新版启动器用户服务 IP 到期阻塞登录链路

> 日期: 2026-05-25

---

## 已确认结论

### 1. 登录失败的根因不是新版启动器登录参数错误，而是用户/认证服务地址失效

- 新版启动器默认用户服务地址仍是：
  - `http://115.231.35.105:3001`
- 旧版启动器使用的也是同一个用户服务地址
- 当前已确认：
  - `115.231.35.105` 到期
  - 因此 `3001` 用户服务链路不可用

### 2. 当前受影响功能

- 登录
- 注册
- 在线玩家列表
- 头像资源 URL（`/avatars/...` 仍拼到 `115.231.35.105:3001`）
- 个人资料接口
- 运行时统计接口
- 在线心跳 / 在线状态上报

### 3. 当前错误表现

- 新启动器登录时报：
  - `登录请求: error sending request for url (http://115.231.35.105:3001/api/login)`
- 当前从开发环境直接探测 `3001` 接口时：
  - `api/login`
  - `api/register`
  - `api/user/online/list`
  均出现远端直接断开连接
- 这与“IP 到期 / 认证服务入口不可达”的判断一致

## 当前阻塞点

### 1. 认证/用户服务地址需要迁移

后续继续开发前，必须先明确新的用户服务地址（替代 `115.231.35.105:3001`），至少需要覆盖：

- `userServerUrl`
- 头像资源基址
- 登录 / 注册 / profile / online-list / heartbeat / runtime-summary 等全部用户接口

### 2. 旧版启动器也依赖同一个用户服务地址

- 旧版启动器不是更换了别的登录协议
- 旧版只是同样直接请求：
  - `userServerUrl/api/login`
- 所以此问题不是“新版特有 bug”，而是“共享用户服务入口失效”

## 下一次继续时的建议顺序

### 步骤1：确认新的用户服务地址

需要明确以下至少一项：

- 新 IP
- 新域名
- 反向代理新入口

以及它是否仍保持：

- `3001 / 3002` 双端口
- 原有 API 路径不变

### 步骤2：统一替换新版启动器中的用户服务基址

需要检查并替换：

- Rust 侧默认 `DEFAULT_USER_SERVER`
- 前端头像拼接地址
- `AppData` 里现有 `config.json` 已保存的旧地址迁移策略

### 步骤3：实机验证完整用户链

至少验证：

- 登录
- 注册
- 在线玩家列表
- 头像加载
- 个人资料弹窗
- 运行时统计
- 在线心跳 / 在线状态刷新
