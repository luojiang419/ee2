# 进度快照 043 — 启动器更新卡死修复与发布链止血逻辑

> 日期: 2026-05-21

---

## 已完成内容

### 1. 启动器本地版本与 release-state 自举逻辑已补齐

- `main.js` 新增本地版本探测链：
  - 优先读取 `release-state.json`
  - 回退 `update/version_history.json`
  - 再回退 `app/package.json`
- 启动器启动时如果缺少 `update/runtime/release-state.json`
  - 会自动补写 bootstrap 状态
  - 不再因为本地 launcher hash 为空而误判必须先自升级

### 2. 启动器自升级失败已支持诊断与熔断

- 新增运行时诊断文件约定：
  - `update/runtime/last-updater-result.json`
  - `update/runtime/last-updater-log.txt`
- 启动器拉起外部更新器时会：
  - 先清理旧诊断文件
  - 传入新的 `--result-file` / `--log-file`
  - 检测更新器是否“启动后立即退出”
- 若上一次 `launcher` 自升级失败且目标版本未变化：
  - 启动器不再 700ms 自动无限重试
  - 弹窗会显示失败原因与日志路径
  - 可手动“重新尝试升级”或“查看日志”

### 3. 外部更新器结果落盘链路已补齐

- `shared/updater_core.py`
  - 成功时写标准结果 JSON
  - 异常时写 `ok=false/error/errorType`
- `updater_gui`
  - GUI 模式和 headless 模式都支持结果文件
  - GUI 日志会同步镜像到文本日志文件

### 4. latest.json 已改为忽略空 launcher 包

- 发布端与 3010 后端现在统一按以下规则决定是否宣告 `packages.launcher`：
  - `launcher/release-manifest.json.files` 非空，或
  - `launcher/release-manifest.json.deleteList` 非空
- 如果 launcher 只是空包：
  - 仍可作为构建产物/历史存档存在
  - 但 `latest.json` 不再暴露 `packages.launcher`

### 5. Flutter 发布端摘要文案已同步

- 本地导出结果现在会明确显示：
  - 本次是否会触发启动器自升级
  - 启动器/游戏删除项数量
- 避免发布者误以为“只要有 launcher 包就一定会触发自升级”

### 6. 本地验证已完成

- `python3 -m py_compile` 已通过：
  - `更新器/src/ee2x_update_suite/...`
  - `更新器/update_backend_mg/app/service.py`
- 临时目录验证通过：
  - `read_release_summary()` 能正确识别空 launcher 包不触发自升级
  - 后端 `_write_latest_json()` 会过滤空 launcher 包
  - 更新器在失败时会正确写出 `last-updater-result.json`
