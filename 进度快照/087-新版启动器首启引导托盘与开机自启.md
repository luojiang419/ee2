# 进度快照 087 — 新版启动器首启引导、托盘驻留与开机自启

> 日期: 2026-05-25

---

## 已完成内容

### 1. 首启与门禁流程改为阻断式引导

- 启动后先检查游戏目录与登录状态
- 目录无效时，强制弹出“选择魔改版游戏目录”引导
- 目录有效但未登录时，强制进入“登录 / 注册”引导
- 登录成功后才进入正常使用状态
- 手动退出登录、登录状态失效后，会重新回到认证引导

### 2. 首页去掉大号选目录占位

- 首页不再显示巨大的“选择游戏目录”标题和按钮
- 首页始终保持正式布局
- `开始游戏` 在目录无效时保持禁用
- 目录缺失场景由阻断式引导弹窗接管

### 3. 设置页新增开机自启，默认关闭行为改为最小化到托盘

- `closeAction` 默认值改为 `minimize`
- 设置页“关闭行为”文案改为“最小化到托盘”
- 新增“开机自启”开关
- 开机自启状态不以本地配置文件为真值源，而是直接读取 / 写入系统自启状态

### 4. Tauri 托盘能力已接入

- 启用 `tray-icon`
- 创建单例托盘图标和托盘菜单
- 托盘菜单提供：
  - 显示主窗口
  - 退出程序
- 点击窗口关闭按钮时，如果配置为 `minimize`，则拦截关闭并隐藏到托盘
- 点击托盘图标左键或托盘菜单“显示主窗口”时，恢复主窗口并聚焦

### 5. 最小化按钮隐藏到托盘补偿逻辑已接入

- 前端轮询当前窗口最小化状态
- 检测到用户点击原生最小化按钮后，自动执行隐藏到托盘
- 解决 Tauri 2 没有直接最小化事件的问题

### 6. 开机自启插件与权限已接好

- 前端新增 `@tauri-apps/plugin-autostart`
- Rust 侧新增 `tauri-plugin-autostart`
- capability 已加入：
  - `autostart:allow-enable`
  - `autostart:allow-disable`
  - `autostart:allow-is-enabled`
- 同时补充了前端隐藏窗口所需的 `core:window:allow-hide`

### 7. 已完成构建与安装包验证

- `npm run build` 通过
- `npm exec tauri build` 通过
- 已重新生成 `1.0.1` NSIS 安装包
