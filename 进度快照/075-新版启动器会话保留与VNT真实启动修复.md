## 已完成内容

### 1. 登录状态改为真正持久化

- 修改文件：
  - `EE2X魔改版启动器/src-tauri/src/lib.rs`
  - `EE2X魔改版启动器/src/App.tsx`
- 修复点：
  - 本地 `user.json` 损坏时自动清理并回退到未登录
  - 启动时如果资料接口失败或联机失败，不再自动 `logout`
  - 只有明确认证失效（`AUTH_INVALID`）时，才清空本地登录状态

### 2. `error decoding response body` 已改为可诊断错误

- 修改文件：`EE2X魔改版启动器/src-tauri/src/lib.rs`
- 已新增统一 JSON 请求包装：
  - 先检查 HTTP 状态码
  - 再检查 `content-type`
  - 最后才做 JSON 反序列化
- 错误现在会带：
  - 请求标签
  - HTTP 状态码
  - `content-type`
  - 响应体前缀
  - 解码阶段

### 3. VNT 改为始终管理员启动，并以真实状态判定成功

- 修改文件：`EE2X魔改版启动器/src-tauri/src/lib.rs`
- 修复点：
  - `VNT` 启动前先清理旧 command-port 标记
  - 改为 PowerShell `Start-Process -Verb RunAs` 管理员启动
  - 启动后轮询真实状态，不再 `sleep 2 秒` 后直接判成功
  - 仅当拿到 `virtual_ip` 时才视为已连接
  - 超时、未生成 command-port、UAC 取消都会返回明确错误
- 网络状态已收口为：
  - `未登录`
  - `未启动`
  - `启动中`
  - `已连接`
  - `启动失败`

### 4. 个人资料弹窗改为“本地会话立即显示 + 远端异步补充”

- 修改文件：
  - `EE2X魔改版启动器/src-tauri/src/lib.rs`
  - `EE2X魔改版启动器/src/App.tsx`
  - `EE2X魔改版启动器/src/lib/types.ts`
  - `EE2X魔改版启动器/src/lib/tauri.ts`
- 修复点：
  - `UserProfile` 新增：

  - 先用本地 `session` 生成基础资料
  - 远端 `profile` 成功则覆盖完整资料
  - `profile` 失败时降级尝试 `runtime summary`
  - 任一路失败都不再让弹窗空白，而是显示局部提示

### 5. 设置页改为内部滚动
