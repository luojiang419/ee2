# 进度快照 089 — 新版启动器首启专用向导与目录下钻

> 日期: 2026-05-26

---

## 已完成内容

### 1. 新增安装后首次运行专用向导模式

- 首次运行不再直接进入正常启动器首页
- 首启流程固定为：
  - 选择魔改版游戏目录
  - 强制更新到最新版本
  - 登录 / 注册并登录
  - 进入正常启动器
- 首启向导完成前，不渲染主启动器首页操作区

### 2. 新增首启状态字段

- 在配置中新增：
  - `setupCompleted`
  - `setupPendingAuth`
- 用于区分：
  - 是否已完成“目录 + 强制更新”
  - 是否在更新完成后等待用户首次登录

### 3. 已支持目录自动向下搜索

- 用户选择的目录不必是最终游戏根目录
- 启动器现在会自动向下查找真正的游戏根目录
- 搜索策略：
  - 广度优先
  - 最大深度 6
  - 最大目录访问数 2000
- 若找到多个候选，会按“深度最浅 + exe 优先级最高 + 路径最短”稳定选中一个

### 4. 首启强制更新接入首启向导

- 当首启目录已确认后，不再进入首页更新中心
- 直接在首启向导内执行一次强制更新
- 更新失败时只允许重试，不允许跳过
- 更新成功后：
  - 写入 `setupCompleted = true`
  - 写入 `setupPendingAuth = true`
  - 然后进入首次登录步骤

### 5. 老用户兼容迁移已接入

- 老配置如果已有有效 `gameDir`
- 启动时会自动补写 `setupCompleted = true`
- 这样旧用户升级后不会被重新拉回首启向导

### 6. 首次登录页面已单独收口为向导页

- 首启登录 / 注册页面不再沿用普通启动器 overlay 体验
- 改为专用向导卡片
- 背景增加模糊层，操作区域居中聚焦
- 登录 / 注册切换按钮和提交按钮都放在下方操作区

### 7. 已完成构建与安装包验证

- `npm run build` 通过
- `npm exec tauri build` 通过
- 已重新生成 `1.0.1` NSIS 安装包
