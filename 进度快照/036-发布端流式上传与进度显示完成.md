# 进度快照 036 — 发布端流式上传与进度显示完成

> 日期: 2026-05-20

## 已完成内容

### 1. 发布端上传链路已改为“先本地打包，再流式上传”

- Flutter 发布端现在仍然先走：
  - `prepare-dual`
- 但上传阶段不再调用 Python `requests files=...` 走整包内存拼装
- 已改成 Flutter 端直接用 `HttpClient`：
  - 从本地 `releaseDir` 读取 manifest / zip
  - 手工构造 multipart/form-data
  - 按块写入网络请求

### 2. 已新增上传进度、百分比、速度显示

- 发布右侧面板新增：
  - `上传进度` 卡片
- 现在上传时会显示：
  - 当前阶段文案
  - 进度条
  - 百分比
  - 已上传大小 / 总大小
  - 实时上传速度
- 发布动作阶段现在分成：
  - `正在本地打包更新内容，请稍候…`
  - `正在上传发布包到服务器…`
  - `上传完成，正在刷新远端状态…`

### 3. 服务端上传链路已改为分块落临时文件

- `更新器/update_backend_mg/app/service.py`
- 现在服务端处理上传包时：
  - manifest 仍读入内存（体积很小）
  - launcher/game zip 不再 `await read()` 全量进内存
  - 改为 1MB 分块写入 `storage/tmp/` 下的临时文件
  - 边写边计算 SHA-256 和总大小
- 只有校验通过后，才复制进正式 `storage/updates/...`
- 发布结束后会自动删除临时目录
