# 进度快照 092 — 新版启动器强制更新全量下载后统一安装

> 日期: 2026-05-26

---

## 已完成内容

### 1. 强制更新现在只同步游戏包

- 已将强制更新范围收口为 `game scope only`
- `launcher` 包不再参与：
  - 首启强制同步
  - 普通启动阻断同步
  - 更新中心手动同步
- 版本统一标准改为只看 `game.version`

### 2. 更新执行模型改为“全部下载后一次安装”

- 先按整条游戏版本链下载所有更新包
- 每个包下载后立即校验哈希
- 全部下载成功后，再按版本顺序统一解压并安装
- 整轮全部完成后才一次性写入 `release_state.game`

### 3. 已彻底移除强制更新后的自动重启依赖

- `restart_required` 固定返回 `false`
- `launcher_stage_ready` 固定返回 `false`
- 前端不再在强制更新后调用：
  - `finalizeUpdateRestart()`
  - `restartSelf()`
- 游戏文件更新完成后直接生效，无需重启启动器

### 4. 已针对死循环根因做收口

- 之前循环的根因是本地 `launcher.version` 永远落后
- 客户端每次都会重新构造 `launcher` 更新链
- 现在已切断这条路径，因此不会再因为 `1.0.18 / 1.0.19` 的 launcher 链路反复更新

### 5. 更新相关文案已同步调整

- 更新中心“版本链”改为“游戏版本链”
- “自动更新当前版本链”改为“自动更新当前游戏版本链”
- 启动阻断弹窗标题改为“同步游戏版本”
- 更新完成弹窗改为显示“当前游戏版本已同步到 …”

### 6. 已完成构建与安装包验证

- `npm run build` 通过
- `npm exec tauri build` 通过
- 已重新生成 `1.0.1` NSIS 安装包
