# 进度快照 090 — 新版启动器首启更新整链完成后统一重启

> 日期: 2026-05-26

---

## 已完成内容

### 1. 修复首启强制更新“更新一个包就重启一次”的问题

- 已确认后端 `run_update_inner` 本身会把整条版本链串起来执行
- 真正的问题在前端：
  - 通用更新函数一拿到 `restartRequired`
  - 就立刻自动执行重启
- 现在已将“执行更新”和“决定是否立刻重启”拆开

### 2. 首启向导更新改为整链完成后统一决定是否重启

- 首启向导更新不再直接复用会自动重启的旧逻辑
- 现在流程变为：
  - 先执行整轮强制更新
  - 整轮更新返回成功结果后
  - 再统一判断是否需要重启
- 如果需要重启，只会在整轮完成后执行一次统一重启

### 3. 普通更新行为保持不变

- 更新中心手动更新仍然保持更新完成后自动重启
- 普通启动时的强制同步更新也仍保持原有自动重启逻辑
- 本次只修正首启向导场景的过早重启问题

### 4. 首启更新状态文案已补充

- 首启更新步骤新增明确状态：
  - “更新包已全部下载并应用，正在重启启动器完成最终同步...”
- 避免用户误判为异常重启

### 5. 已完成构建与安装包验证

- `npm run build` 通过
- `npm exec tauri build` 通过
- 已重新生成 `1.0.1` NSIS 安装包
