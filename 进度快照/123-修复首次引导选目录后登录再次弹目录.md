## 已完成内容

### 1. 已定位根因

- 问题表现：
  - 首次引导页已经选择游戏目录
  - 强制更新完成后进入登录页
  - 登录/注册成功后又再次弹出“选择游戏根目录”
- 根因不是后端，也不是路径校验规则本身
- 根因是前端状态竞争：
  - 选择目录后，界面先 `setConfig(next.config)`
  - 紧接着首次引导强更完成时调用 `saveSetupFlags`
  - 旧实现里 `saveSetupFlags` 直接使用当前 React 闭包中的 `config`
  - 该 `config` 在同一轮调用里仍可能是“选目录前的旧值”
  - 结果把刚刚选好的 `gameDir / gameExePath` 用空值覆盖回配置文件
- 随后登录成功刷新 bootstrap 时：
  - `setupCompleted=true`
  - 但 `config.gameDir` 已被覆盖为空
  - 最终再次进入选目录流程

### 2. 已完成代码修复

- 已修改：
  - `EE2X远航版启动器/src/App.tsx`
  - `EE2X魔改版启动器/src/App.tsx`
- 修复方式：
  - `saveSetupFlags` 新增 `baseConfig` 参数
  - 优先使用调用方显式传入的最新配置快照，而不是盲信当前 React 状态闭包
  - 首次引导强更完成时，显式传入 `currentBoot.config`
  - 登录/注册完成后，继续显式传入当前 `config`
