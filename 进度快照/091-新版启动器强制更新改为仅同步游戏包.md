# 进度快照 091 — 新版启动器强制更新改为仅同步游戏包

> 日期: 2026-05-26

---

## 已完成内容

### 1. 强制更新逻辑改为只同步游戏包

- 现在强制更新只处理 `game scope`
- `launcher scope` 不再参与：
  - 首启强制同步
  - 普通启动强制同步
  - 更新中心手动强制同步
- 版本统一标准改为只看 `game.version`

### 2. 修复 1.0.18 / 1.0.19 循环更新问题

- 根因已确认：
  - 本地 `launcher.version` 一直落后
  - 客户端每次都会继续构造 launcher 更新链
  - 再叠加此前自动重启，导致反复进入同一批版本更新
- 现在已彻底切断：
  - 不再消费 `launcher` 包
  - 不再因 `launcher.version` 落后而阻断或重启

### 3. 更新执行模型改为“全部下载后一次安装”

- 先按整条游戏版本链下载全部更新包
- 每个包下载后立即校验 `sha256`
- 全部下载成功后，再按发布时间顺序统一解压并应用到游戏目录
- 整轮完成后才统一写入 `release_state.game`
- 避免中途落盘导致断点误判和链路混乱

### 4. 更新成功后不再重启启动器

- `restart_required` 固定返回 `false`
- `launcher_stage_ready` 固定返回 `false`
- 前端已移除强制更新后的自动重启依赖
- 游戏文件更新完成后直接生效，无需重启启动器

### 5. UI 文案已同步调整

- 更新中心“版本链”改为“游戏版本链”
- “自动更新当前版本链”改为“自动更新当前游戏版本链”
- 强制同步相关弹窗文案改为“同步游戏版本”
- 更新完成弹窗改为显示“当前游戏版本已同步到 …”

### 6. 已完成构建与安装包验证

- `npm run build` 通过
- `npm exec tauri build` 通过
- 已重新生成 `1.0.1` NSIS 安装包
