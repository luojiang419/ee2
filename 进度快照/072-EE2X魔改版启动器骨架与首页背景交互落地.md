# 进度快照 072 — EE2X魔改版启动器骨架与首页背景交互落地

## 已完成内容

### 1. 新项目目录与技术骨架已落地

- 新建独立项目目录：`EE2X魔改版启动器/`
- 技术栈：`Tauri v2 + React + TypeScript + Vite`
- 已落地基础文件：
  - `package.json`
  - `vite.config.ts`
  - `tsconfig*.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/tauri.conf.json`
- 已复制桌面端资源：
  - `src-tauri/icons/EE2X.ico`
  - `src-tauri/resources/vnt/vnt-cli.exe`
  - `src-tauri/resources/vnt/wintun.dll`

### 2. 首页布局已按当前确认方案实现

- 左侧固定为在线玩家公示栏
- 顶部栏已实现：
  - 标题
  - 网络状态
  - 虚拟网卡上下行速度（1s 刷新）
  - 版本号显示（格式：`版本 v1.0.15`）
  - 头像/登录区
- 首页右侧操作区已锁定为纵向堆栈：
  - 联机状态
  - 一键更新
  - 游戏设置
  - 开始游戏
- 已修正右侧按钮与开始游戏按钮的贴合关系，当前布局正确
- 未选择路径时，首页中央显示：
  - `选择 Empire Earth II 游戏根目录`
- 已选择有效路径后：
  - 路径显示在左下角
  - 开始游戏保留在右下角

### 3. 浏览器预览模式 fallback 已补齐

- `localhost:1420` 浏览器预览不再只看静态界面
- 已在 `src/lib/tauri.ts` 中补充 mock/fallback：
  - 选择游戏目录
  - 登录 / 注册 / 登出
  - 在线玩家列表
  - TUN 联机状态
  - 顶部网络速率滚动
  - 更新检查与链式更新进度模拟
- 已修复浏览器旧配置缺字段导致“游戏设置页空白”的问题

### 4. 背景功能已接入第一版

- 设置页已支持背景类型：
  - `默认背景`
  - `背景图片`
  - `背景视频`
- 交互方式已改成类似旧版启动器：
  - 先选背景类型
  - 再显示对应的“浏览”入口
- 已增加背景预览卡片
- 已增加“恢复默认背景”按钮
- 已接入首页图片/视频全屏铺底显示
- Tauri 桌面端已增加 `import_background_media`：
  - 选中的图片/视频会导入启动器自己的本地数据目录
  - 不再只依赖原始文件路径

### 5. Rust 侧基础能力已接入

- `src-tauri/src/lib.rs` 已实现基础命令：
  - 配置读写
  - 游戏根目录校验
  - 启动游戏
  - 登录 / 注册 / 资料读取
  - 在线玩家获取
  - TUN-only VNT 启停与状态查询
  - 在线状态上报
  - 更新检查 / 更新执行 / 重启收尾
  - 背景媒体导入
- 已明确只走 TUN，不再使用 TAP

### 6. 与旧服务兼容的协议已接线

- 账号服务继续使用旧 `3001`
- 更新服务继续使用旧 `3010`
- 更新链路保留：
  - `latest`
  - `history`
  - `ws` 推送
- 更新完成后已改为弹窗反馈，不在首页常驻更新状态卡

### 7. 当前验证结果

- `vite build`：通过
- `cargo check`：通过
- 浏览器预览地址：
  - `http://localhost:1420/`
