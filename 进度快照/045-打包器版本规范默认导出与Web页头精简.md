# 进度快照 045 — 打包器版本规范默认导出与Web页头精简

> 日期: 2026-05-21

---

## 已完成内容

### 1. Flutter 更新包打包器版本号规范化

- 版本号默认基线改为 `1.0.0`
- 新增补丁位 `- / +` 按钮
  - `+`：仅增加补丁位
  - `-`：仅减少补丁位，不借位，不低于 `1.0.0`
- 输入规范统一为 `x.y.z`
  - `1` → `1.0.0`
  - `1.2` → `1.2.0`
  - `v1.2.3` → `1.2.3`
  - 兼容全角点号
- Python 共享层新增统一版本工具，bridge 与本地打包链路共用

### 2. 本地导出规则改为默认保存到 `data/zip`

- 不再先弹保存对话框
- 默认保存目录：
  - 打包后的 EXE：工具同级 `data/zip`
  - 源码运行：`更新器/data/zip`
- 默认文件名改为：
  - `ee2x-{version}-{YYYYMMDD-HHmmss}.zip`
- 最近一次导出结果卡片新增 `另存到...`
  - 基于默认保存副本执行复制
  - 不重新打包

### 3. 3010 Web 推送页页头精简

- 移除可见的 `固定频道 stable` 卡片
- `当前账号 / 退出登录` 模块保留并移动到原右上角位置
- `data-channel` 与前端内部频道逻辑保留，未改后端接口
- 同步移除 `channelLabel` DOM 依赖，避免控制台报错

### 4. 文档与检查

- 已更新 `更新器/docs/发布协议.md` 的默认导出目录与命名说明
- 已完成检查：
  - `python3 -m compileall 更新器/src/ee2x_update_suite 更新器/update_backend_mg/app`
  - `flutter analyze`
  - Python 版本规范样例自测
