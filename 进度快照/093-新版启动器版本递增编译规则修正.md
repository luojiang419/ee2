> 日期: 2026-05-26

## 已完成内容

### 1. 已定位“版本号没递增但源码版本已变化”的根因

- `tauri build` 会先读取 `src-tauri/tauri.conf.json` 的版本号
- 然后才执行 `beforeBuildCommand`
- 因此如果把版本递增放进 `beforeBuildCommand`
- 本次编译虽然会改源码版本，但安装包文件名仍然会沿用旧版本号

### 2. 已修正新版启动器正式编译入口

- 正式编译命令恢复并固定为：
  - `npm run build:release`
- 执行顺序现在为：
  - 先执行 `bump:version`
  - 再执行 `tauri build`
- 这样 `tauri build` 在启动时就能直接读取递增后的新版本号

### 3. 已恢复 `beforeBuildCommand` 只负责前端构建

- `src-tauri/tauri.conf.json` 中的 `beforeBuildCommand` 已恢复为：
  - `npm run build:frontend`
- 不再在这里做版本号递增

### 4. 已补充项目总规则

- 在 `地球帝国游戏开发规则.md` 中新增并明确：
  - 新版启动器每次编译安装包时版本号必须自动 `+0.0.1`
  - 正式编译入口固定为 `npm run build:release`
  - 严禁再直接使用 `npm exec tauri build` 绕过版本递增

### 5. 已重新编译并验证版本号真实递增

- 重新执行 `npm run build:release`
- 已确认版本源同步递增为：
  - `package.json` → `1.0.3`
  - `tauri.conf.json` → `1.0.3`
  - `Cargo.toml` → `1.0.3`
- 已生成新安装包：
  - `EE2X魔改版启动器_1.0.3_x64-setup.exe`
