# EE2X 更新服务器部署说明

## 当前线上结构

当前用于启动器更新的服务不是文档里写的旧目录，而是：

- 服务根目录：`/opt/ee2x_up-xg`
- 运行方式：`pm2`
- 进程名：`ee2x-xg`
- 主程序：`/opt/ee2x_up-xg/main.py`
- 对外端口：`1234`

另有一套旧服务：

- 目录：`/opt/ee2x/ee2x_up`
- 端口：`1888`

本次重构接入的是 `1234` 这套。

## 本次新增的静态发布入口


## 生产发布建议

1. 先用包工具生成正式版本 ZIP 和 manifest
2. 核对版本号、更新说明、删除清单
3. 用 `publish.local.json` 发布到 `stable`
4. 发布后在另一台机器上验证：
   - `latest.json`
   - manifest
   - ZIP 下载
   - 启动器一键更新
